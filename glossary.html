Glossary

Every technical term referenced in the topic pages, with a short definition and back-links to the pages where it's used in depth. Strict scope — only terms that actually appear in the notes.

ABCDEF GHIJKL MNOPQR STUVWX YZ
A
access latency
How long you wait before the first byte arrives. Set by the device's physics — separate from bandwidth, which is how fast bytes flow once they start.
Used in:Latency
A record
A DNS record that maps a name directly to an IPv4 address — the terminal answer in a resolution chain. AAAA is the same shape for IPv6.
Used in:DNS Resolution
ALIAS record ANAME · CNAME flattening
A provider-side trick (Route 53 ALIAS, Cloudflare CNAME flattening, DNSimple ALIAS, NS1 ANAME) that lets you point an apex domain at another name. The provider resolves the target at query time and serves the resulting IPs as a real A record, sidestepping the rule that the apex can't be a CNAME.
Used in:DNS Resolution
API gateway
A single public endpoint that fronts many backend services and centralises cross-cutting concerns: authentication, rate limiting, routing, observability, API versioning. Backends sit behind it as plain services. Common implementations: Kong, AWS API Gateway, Apigee, Envoy, Tyk.
Used in:The API Gateway PipelineAPI Design Best Practices
API versioning
Strategy for evolving an API without breaking integrated clients. Three common shapes: URL versioning (/v1/users) — explicit and easy to debug, but the segment lives in every URL forever; header versioning (API-Version: 2 or Accept: application/vnd.acme.v2+json) — clean URLs but invisible from curl; date-based versioning (Stripe's Stripe-Version: 2024-04-01) — header-style with calendar semantics. Pair with the Deprecation and Sunset response headers so clients know a cutover is coming before it lands.
Used in:API Design Best Practices
apex root domain · naked domain · zone apex
The zone name itself — example.com, not www.example.com. The apex must carry mandatory SOA and NS records, which is why you can't put a CNAME there: CNAME refuses to coexist with any other record.
Used in:DNS Resolution
asymmetric crypto public-key cryptography
Cryptographic schemes where each party has two mathematically linked keys: a public one given out freely and a private one kept secret. Encrypt with public, decrypt with private; sign with private, verify with public. Slow compared to symmetric ciphers (AES), so used sparingly — typically just to bootstrap a shared secret.
Used in:HTTPS & The TLS Handshake
async 202 handoff long-running operation pattern
The pattern for operations that can't fit in one request/response cycle: server accepts the work and returns 202 Accepted immediately with a Location header pointing at a status endpoint. Client polls the status endpoint (or registers a webhook) until the job finishes. On completion, the status endpoint redirects to the result with 303 See Other. Survives gateway timeouts, dropped connections, and client crashes — the job ID is the resumable handle.
Used in:API Design Best Practices
authoritative DNS authoritative name server
The name server that holds the actual zone file — the source of truth for some slice of names. Doesn't recurse; just answers definitively about its own zone. Distinct from a recursive resolver, which is a cache that walks the authoritative chain on the client's behalf.
Used in:DNS ResolutionDNS TTL & Cutovers
AOF append-only file
Redis's optional durability log. Without it, Redis is an in-memory cache and is lost on restart; with it, every write is appended to disk so state can be replayed.
Used in:Memory Hierarchy
AoS array of structs
Record-major memory layout: {x,y,z,name}, {x,y,z,name}, .... Best when a loop touches all fields of one record at a time.
Used in:Cache Locality
B
backoff
A retry policy where each successive attempt waits longer than the last (typically exponentially: 100 ms, 200 ms, 400 ms…). Gives the downstream room to recover and prevents busy-wait amplification.
Used in:Resilience Patterns
bcrypt
A password-hashing function that's deliberately slow — the slowness is the security feature. The cost factor controls how many rounds; bcrypt cost 12 takes ~250 ms on a single core.
Used in:Hardware Pressure
bottleneck
The component that fills up first under load. System throughput is the bottleneck's throughput — optimizing anything else is zero-impact until you move the bottleneck.
Used in:Hardware PressureThroughput
branch prediction
The CPU's bet on which side of an if will be taken, made before the condition can be computed. >95% accurate on regular data; coin-flips on random data, where every miss costs a 10–20 cycle pipeline flush.
Used in:Cache Locality & Branch Prediction
bulkhead
Partitioned resources (separate thread pools, connection pools, lanes) so that a failure in one downstream can't drain pools used by others. Named after the watertight walls in a ship's hull — contains a breach to one compartment.
Used in:Resilience Patterns
C
C10K problem
The late-1990s challenge of handling 10,000 concurrent connections on a single server. Killed the "thread per connection" model and drove the move to event loops and lightweight threads.
Used in:Context Switching
cache line
The smallest unit of memory traffic between CPU caches and DRAM — almost universally 64 bytes. Reading one byte pulls in the surrounding 63 at the same cost.
Used in:Cache LocalityMemory HierarchyHardware Pressure
cache stampede
When a hot cache key expires and many requests miss simultaneously, all hammering the database to rebuild it. Sometimes called "dog-piling." Mitigated by single-flight, probabilistic refresh, stale-while-revalidate, or TTL jitter.
Used in:Resilience Patterns
CDN content delivery network
A network of edge servers that cache content close to viewers, spreading bandwidth and latency burden across many geographically-distributed boxes instead of one origin.
Used in:Hardware Pressure
certificate SSL cert · TLS cert · X.509
A signed document binding a public key to an identity (a domain name, typically). Issued by a Certificate Authority and verified by clients against a trust store of known CAs. The browser shipping ~150 root CAs is what makes "https://bank.com" actually identify bank.com.
Used in:HTTPS & The TLS Handshake
Certificate Authority CA
An entity that issues and signs certificates. Browsers and operating systems ship with a hardcoded list of root CAs they trust (Let's Encrypt, DigiCert, GlobalSign, etc.); a cert signed by one of these is automatically accepted, while a self-signed cert is not. The trust store is the root of the chain.
Used in:HTTPS & The TLS Handshake
CNAME canonical name
A DNS record that aliases one name to another. Resolvers follow the chain (CNAME → CNAME → … → A) until they reach a terminal record. Two rules: a name with a CNAME can have no other records, and the apex can't be a CNAME (because SOA and NS are required there).
Used in:DNS Resolution
circuit breaker
A wrapper around a remote call that tracks failure rate. When failures cross a threshold, the breaker "opens" and fails subsequent calls instantly (no downstream contact). After a cooldown, a probe request decides whether to close again.
Used in:Resilience Patterns
coherence cache coherence
The property that all cores see consistent values for shared memory. Maintained by a hardware protocol that invalidates or updates other cores' cached copies whenever one core writes.
Used in:Memory Hierarchy
content negotiation
The HTTP mechanism by which a client and server agree on the response representation. The client sends an Accept header listing acceptable media types (e.g., application/json, application/xml); the server picks the best match it supports and returns it, or responds with 406 Not Acceptable. Most APIs default to JSON when no preference is given.
Used in:API Design Best Practices
consensus
A class of distributed algorithms (Paxos, Raft, Zab) where a majority of nodes must agree before a value is committed. The basis of CP systems — minority partitions can't make progress.
Used in:CAP Theorem
CORS Cross-Origin Resource Sharing
The browser protocol that relaxes the same-origin policy so JavaScript on one origin can call an API on another. Simple requests (GET/HEAD/form-encoded POST) carry an Origin header and the server replies with Access-Control-Allow-Origin; non-simple requests (custom headers, PUT/DELETE/PATCH, JSON bodies) trigger an OPTIONS preflight first. Caching the preflight via Access-Control-Max-Age turns the two-round-trip cost back into one.
Used in:API Design
copy-on-write COW
A trick that makes fork() cheap: pages are shared read-only between parent and child until one of them writes, at which point only that page is duplicated. Most fork-exec patterns never trigger duplication at all.
Used in:Process vs Thread
CPU affinity core pinning
A per-thread bit-mask that restricts which cores the kernel scheduler can run that thread on. Pinning narrows the mask — sometimes to a single core — to keep cache state warm and avoid migration.
Used in:Core Pinning
cursor pagination keyset pagination · seek pagination
A pagination scheme where each response includes an opaque cursor — typically the sort key of the last row — that the client passes back to fetch the next page. The query becomes "rows after this cursor" rather than "skip N rows", which is O(log N) per page and stable under concurrent writes. Trade-off: no random-access "page 50".
Used in:PaginationAPI Design Best Practices
D
DDR5
The current generation of DRAM signalling standard. Roughly 100 GB/s of bandwidth per channel on modern server DIMMs, ~80 ns access latency.
Used in:Memory Hierarchy
Diffie-Hellman DH · DHE · ECDHE
A key-exchange protocol that lets two parties on a hostile network derive the same shared secret without ever sending it. Each side picks a private value, exchanges a public derivation, and combines them to land on gab mod p. An eavesdropper sees both public values but recovering a or b is the discrete log problem — infeasible at large p.
Used in:HTTPS & The TLS Handshake
dig
The standard DNS lookup tool. dig name uses the system resolver; dig name @8.8.8.8 queries a specific resolver; dig name @ns1.example.com queries an authoritative server directly, bypassing every cache — the move for "did my change actually land?". +short trims the output; +trace shows the full recursion.
Used in:DNS TTL & Cutovers
DNS Domain Name System
The internet's name-to-IP translation system, structured as a tree of authoritative servers fronted by a forest of caches. A query falls through five cache layers (browser, OS, router, recursive resolver, authoritative) — the first non-expired copy wins.
Used in:DNS ResolutionDNS TTL & Cutovers
discrete logarithm problem DLP
Given g, p, and y = gx mod p, find x. Computing gx mod p is fast; reversing it is infeasible at large p. That asymmetry is the foundation of Diffie-Hellman key exchange and several digital-signature schemes.
Used in:HTTPS & The TLS Handshake
DRAM dynamic RAM
The main-memory tier. One transistor + one capacitor per bit; cheap and dense, but has to be periodically refreshed and lives off-chip from the CPU. ~80 ns access latency.
Used in:Memory HierarchyCache Locality
E
ECDHE elliptic-curve Diffie-Hellman ephemeral
Diffie-Hellman over an elliptic-curve group instead of integers mod a prime. Same protocol structure, but equivalent security at smaller key sizes (256 bits ≈ 3072-bit RSA) and faster operations. The "ephemeral" half means a fresh keypair is generated for each session and discarded — the basis for forward secrecy.
Used in:HTTPS & The TLS Handshake
eventual consistency
A consistency model where writes are accepted by any node and asynchronously replicated; replicas may diverge during a partition but converge after it heals. The default for AP systems.
Used in:CAP Theorem
F
false sharing
When two independent variables happen to live on the same 64-byte cache line, two cores writing them appear to the coherence protocol as conflicting on a single line — and ping-pong it back and forth. Fix: pad to cache-line boundaries.
Used in:Memory Hierarchy
forward proxy
A proxy that sits in front of clients and represents them to destination servers — the mirror image of a reverse proxy. Common uses: corporate egress filtering, outbound caching, audit/DLP logging, anonymity / IP masking (Tor, VPN), and geo-routing. Mnemonic: forward proxy hides the client; reverse proxy hides the server.
Used in:The API Gateway Pipeline
forward secrecy perfect forward secrecy · PFS
A property of a key-exchange protocol: even if a long-term secret leaks tomorrow, traffic recorded today stays unreadable. Achieved by deriving each session's encryption key from ephemeral values that are discarded when the session ends. TLS 1.3 mandates it; old RSA key exchange (TLS 1.2 and earlier) didn't have it.
Used in:HTTPS & The TLS Handshake
fsync
The system call that forces a write all the way to durable storage. Without it, the OS may report "done" as soon as bytes hit a buffer; fsync is what every database calls before acknowledging a commit. ~10–50 µs on NVMe.
Used in:Hardware PressureMemory Hierarchy
G
GIL Global Interpreter Lock
A runtime lock (most famously Python's) that serializes bytecode execution across threads. Threads still help for I/O-bound work because the GIL releases on syscalls, but CPU-bound parallelism requires multiple processes.
Used in:Process vs Thread
goodput
Throughput minus errors, retries, and duplicates — only the useful work. When a system gets slow, throughput and goodput diverge; that gap is a signal of failure amplification.
Used in:Throughput
gossip
A peer-to-peer failure-detection protocol where every node periodically pings a random subset of peers and shares state. Convergent and resilient; the way most distributed clusters know who's alive.
Used in:Consistent Hashing
graceful degradation
Returning a useful but reduced response when a dependency is down — global popular list instead of personalized, "price unavailable" instead of an error page. The fallback is part of the spec, not an accident.
Used in:Resilience Patterns
GraphQL
A query language for APIs. The client sends a query naming exactly the fields it wants; the server responds with a matching shape. Solves over-fetching and N+1 round-trips at the cost of URL-based caching, by-count rate-limiting, and resolver-side N+1 query problems (mitigated by DataLoader). Best fit: BFF layers aggregating many backends for one UI.
Used in:API Design
green thread goroutine · virtual thread · fiber
A user-space thread scheduled by a language runtime onto a small pool of OS threads. Switch cost is hundreds of nanoseconds instead of microseconds. Examples: Go goroutines, Java virtual threads, async Rust tasks, Erlang processes.
Used in:Process vs ThreadContext Switching
gRPC
Google's modern RPC framework — .proto schemas (Protocol Buffers), HTTP/2 transport, generated client/server stubs in many languages, and built-in support for unary, server-streaming, client-streaming, and bidirectional-streaming calls. Best fit: internal service-to-service where you control both ends and want speed plus typed contracts.
Used in:API Design
H
hash ring
A circular keyspace (typically 0 to 232−1) where both servers and keys are placed by hash. To find a key's owner, walk clockwise from the key's position until you bump into a server.
Used in:Consistent Hashing
HATEOAS Hypermedia as the Engine of Application State
The REST property where each response embeds hyperlinks to the actions a client can take next, so the client navigates by following links instead of hard-coding URL templates. Roy Fielding's original dissertation argues this is what separates REST from "JSON over HTTP"; in practice most public APIs ship URI templates in their docs and skip it. Variants that lean into it: HAL, JSON:API, Siren.
Used in:API Design
HBM high-bandwidth memory
DRAM stacked vertically and bonded to the same package as the processor. Used on modern GPUs and AI accelerators. Plays the role of "main memory" in those architectures.
Used in:Memory Hierarchy
hedged request
If the first replica hasn't answered by the p95 latency, send a copy of the request to a second replica and take whichever returns first. Trades a little extra load for a dramatic improvement in tail latency.
Used in:Latency & Throughput
HLS HTTP Live Streaming
Apple's video-streaming protocol where a stream is broken into small (~2–10 second) MP4 chunks and a manifest lists them. Each chunk is fetched as a normal HTTP request, which lets streams flow through any HTTP CDN.
Used in:Hardware Pressure
HTTP method HTTP verb
The action component of an HTTP request: GET reads, POST creates, PUT replaces, PATCH partially updates, DELETE removes. HEAD and OPTIONS are auxiliary. Each verb has an idempotency and safety property baked into the spec — worth memorising because retries and caching depend on them.
Used in:API DesignAPI Design Best Practices
I
idempotency idempotent
A property where running the same operation twice has the same effect as running it once. Reads are naturally idempotent; mutations need an explicit idempotency key (a UUID the client generates and the server dedupes by). Precondition for safe retries.
Used in:Resilience PatternsAPI DesignAPI Design Best Practices
idempotency key
A client-generated unique token (typically a UUID) sent on a mutation request. The server stores the response keyed by it and returns the cached response on retry, so the same logical request can be sent ten times and execute exactly once. Stripe popularised it; standard for payment APIs. Pair with a TTL longer than the client's retry budget.
Used in:API DesignAPI Design Best Practices
IOPS I/O operations per second
How many discrete read/write operations a disk can sustain. A different number for sequential vs random, small vs large blocks. NVMe SSDs do hundreds of thousands; spinning disks do ~150.
Used in:Hardware Pressure
IPC inter-process communication
Mechanisms for two processes to exchange data across address-space boundaries — pipes, sockets, shared memory regions, message queues. Slower than two threads sharing a pointer; that's the whole point.
Used in:Process vs Thread
isolcpus
A Linux kernel boot parameter that excludes specific cores from the general scheduler's load-balancing pool. Used with nohz_full and pinning to dedicate cores entirely to latency-critical threads.
Used in:Core Pinning
J
jitter
Randomness added to retry delays or scheduled events to prevent synchronized waves. Variants — equal, full, decorrelated — control how much randomness vs predictability you keep.
Used in:Resilience Patterns
K
knee (of the curve)
The point on the latency-vs-load curve where latency starts climbing steeply with small increases in offered load. Operating left of the knee is healthy; operating near it is risky.
Used in:Latency & Throughput
L
leaky bucket
A throttling algorithm: requests enter a fixed-size queue and drain out at a constant rate. Output is perfectly smooth no matter how spiky the input.
Used in:Resilience Patterns
libuv
The C library Node.js uses to implement its event loop. Handles I/O, timers, file reads, and DNS lookups across an OS-thread pool, calling back into the JS engine when results are ready.
Used in:The Event Loop
line rate
The maximum byte throughput a network interface can sustain — the bandwidth ceiling of a NIC. A 10 Gbps NIC has a line rate of ~1.25 GB/s of payload.
Used in:Hardware Pressure
linearizability
The strictest consistency guarantee: every read sees the most recent write or an error. Operations appear to take effect at a single instant between their request and response.
Used in:CAP Theorem
Little's Law
L = λ × W — for any stable system, average concurrency equals arrival rate times average residence time. Three numbers; fix any two and the third is forced. The most useful equation in performance engineering.
Used in:Latency & ThroughputBack-of-the-Envelope
load balancer L4 LB · L7 LB
A box that spreads requests across the N instances of a single service. L4 balances at the transport layer (TCP packets, 5-tuple hash) — fast and protocol-agnostic. L7 parses HTTP and balances by URL, header, or cookie — slower but content-aware. Algorithms: round-robin, least-connections, least-response-time, consistent-hashing, weighted.
Used in:The API Gateway Pipeline
M
macrotask
A task on the JavaScript event loop's main queue — setTimeout, setInterval, I/O completions, DOM events. The loop runs exactly one macrotask per tick, then drains all microtasks.
Used in:The Event Loop
MESI
The classic four-state cache-coherence protocol — every cache line is in one of four states: Modified, Exclusive, Shared, or Invalid. Variants (MOESI, MESIF) add states but follow the same shape.
Used in:Memory Hierarchy
microtask
A higher-priority task queued by Promise.then, queueMicrotask, or MutationObserver. Microtasks drain entirely between every macrotask — Promise.then always runs before the next setTimeout.
Used in:The Event Loop
mTLS mutual TLS · client-authenticated TLS
A TLS variant where both sides — not just the server — present certificates. The server verifies the client's cert against its own trust store. Used inside zero-trust infrastructure and B2B APIs, but not on the public web (browsers don't have certs).
Used in:HTTPS & The TLS Handshake
N
network partition
When the network splits a cluster into groups that can't reach each other. Not optional in real systems — partitions happen, and CAP says you have to choose what to do during them.
Used in:CAP Theorem
NIC network interface card
The physical network port. Has two distinct limits — bandwidth (Gbps) and packets-per-second — that often saturate independently.
Used in:Hardware Pressure
NUMA Non-Uniform Memory Access
A multi-socket architecture where DRAM is split across sockets. Local-socket access is fast (~80 ns); remote-socket access goes over the inter-socket link (~120–160 ns). Pinning keeps thread and memory on the same NUMA node.
Used in:Core Pinning
O
OAuth 2.0 OAuth2 · OIDC
The standard authorisation framework for APIs. The client gets an access token from an authorisation server and presents it on each request; the resource server verifies the token's signature, expiry, scope, and audience before honouring the call. Grant types match client shapes: Authorization Code + PKCE (web/mobile), Client Credentials (service-to-service), Device Code (TVs/CLIs). OpenID Connect (OIDC) layers identity (the "who") on top of OAuth's authorisation (the "what are you allowed to do").
Used in:API Design Best Practices
offset pagination offset/limit
A pagination scheme where the client requests limit=N&offset=K and the server returns rows K+1 through K+N. Simple and supports random-access ("page 50 of 200"), but two failure modes: cost grows linearly with offset (deep pages are slow), and the result is inconsistent under concurrent writes (rows duplicate or skip).
Used in:PaginationAPI Design Best Practices
OpenAPI Swagger
A YAML or JSON specification format describing every endpoint of an HTTP API — paths, methods, parameters, request and response schemas, error envelopes, auth requirements. One spec drives an ecosystem: interactive docs (Swagger UI, Redoc), generated client SDKs, server-side request validation, mock servers, contract tests. Two authoring styles: spec-first (write YAML, generate stubs) or code-first (annotate the implementation, generate spec). Either works as long as one is the source of truth.
Used in:API Design Best PracticesAPI Design
over-fetching
An API returning more fields than the client needs. Common in REST when one endpoint serves many call sites with different needs. Wastes bandwidth and parse time, especially on mobile. GraphQL solves it by letting the client name the fields it wants.
Used in:API Design
P
PACELC
Daniel Abadi's extension of CAP: if there's a Partition, choose A or C; Else (normal operation), choose Latency or Consistency. Captures the trade-off CAP misses — even without partitions, consistency costs latency.
Used in:CAP Theorem
page cache
The kernel's RAM cache of recently-read disk pages. Why "the second read of the same file" is much faster than the first — and why a cold cache after deploy creates spikes in latency.
Used in:Hardware Pressure
PCB Process Control Block
A per-thread kernel struct in RAM that holds saved registers, program counter, stack pointer, and scheduling state for a paused thread. Every context switch is a CPU↔RAM round-trip through the PCB.
Used in:Process vs Thread
peak factor
The multiplier from average load to peak load — typically ×3 for general consumer apps, ×5 for clearly diurnal workloads, ×2 for global services. You capacity-plan against peak; you bill against average.
Used in:Back-of-the-Envelope
percentile
A latency distribution marker: p95 = 120 ms means 95% of requests finished in ≤120 ms. p99, p99.9, p99.99 reach into the tail. Percentiles describe what users experience; averages hide the tail.
Used in:Latency & Throughput
pipeline CPU pipeline · instruction pipeline
The CPU's overlapping queue of in-flight instructions, typically 15–20 stages deep. A branch misprediction means flushing every speculative instruction, costing 10–20 cycles.
Used in:Cache Locality & Branch Prediction
pps packets per second
The other network limit, separate from bandwidth. A 10 Gbps NIC tops out at ~1–2 million pps before per-packet OS overhead bites — so a flood of tiny requests can saturate pps long before bytes.
Used in:Hardware Pressure
preemption
An involuntary context switch — the kernel forcibly takes the CPU away from a running thread, typically because its time slice expired or a higher-priority thread became runnable.
Used in:Context Switching
prefetcher
CPU hardware that watches access patterns and pulls in cache lines before the program asks for them. Sequential access is its happy path; random access defeats it entirely.
Used in:Cache Locality
Problem Details RFC 7807 · application/problem+json
A standard JSON shape for HTTP error responses defined by RFC 7807. Returns a media type of application/problem+json and a body with type, title, status, detail, and instance fields, plus any extensions (code, hint, trace_id). Lets clients write one error-handling code path against every endpoint instead of one per service.
Used in:API Design Best Practices
Protobuf Protocol Buffers
Google's schema-first IDL for binary message serialisation. You write a .proto file describing messages and services; a compiler generates type-safe stubs in many languages. Wire format is tag-and-varint — ~3× smaller than the equivalent JSON. The serialisation layer underneath gRPC.
Used in:API Design
Q
quorum
The number of replicas that must acknowledge a read or write. With replication factor N, the threshold R + W > N guarantees linearizability. Cassandra and DynamoDB tune this per query.
Used in:CAP Theorem
R
race condition
When the result of an operation depends on the unpredictable interleaving of two or more threads accessing shared state. The hazard is in contested data, not contested cycles — locks/atomics exist to manage it.
Used in:Process vs Thread
Raft
A consensus algorithm designed to be more understandable than Paxos. Used by etcd, Consul, CockroachDB, TiKV, and many other modern distributed systems. Elects a leader; followers replicate the leader's log.
Used in:CAP Theorem
range request byte range · partial content
An HTTP mechanism for fetching a slice of a large resource instead of the whole thing. The server advertises support with Accept-Ranges: bytes; the client asks with a Range: bytes=0-1048575 header; the server replies with 206 Partial Content and a Content-Range echoing what was returned. Enables resumable downloads, parallel chunked fetches, and video-style seeking. Distinct from pagination, which is for collections; ranges are for single large resources.
Used in:API Design Best Practices
rate limiting
Capping how many requests a given identity (user, API key, IP) can issue per window. Two purposes: fairness (one tenant can't starve others) and defense (cap the cost of abuse or DoS). Lives at the gateway because the gateway sees every request — putting it deeper means costs are paid before the limit kicks in. Common algorithms: token bucket, leaky bucket, fixed-window, sliding-window log.
Used in:The API Gateway PipelineAPI Design Best Practices
recursive resolver
A DNS resolver that, on a cache miss, walks the authoritative chain on the client's behalf — root → TLD → authoritative — and caches every step on the way back. Examples: 8.8.8.8 (Google), 1.1.1.1 (Cloudflare), your ISP's resolver. Distinct from an authoritative server, which doesn't recurse but holds the actual zone file.
Used in:DNS ResolutionDNS TTL & Cutovers
replication factor RF
How many copies of each piece of data the cluster keeps — typically 3. Production systems also place those copies on distinct racks or availability zones so that a single rack failure doesn't take all replicas.
Used in:Consistent Hashing
REST Representational State Transfer
A resource-oriented API style: URL identifies a resource, HTTP verb describes the action on it. Stateless (each request is self-contained), cacheable via HTTP semantics, text-based (usually JSON), and loosely coupled (URLs are the contract, not generated stubs). Dominant on the public web for the universality and tooling, slower on the wire than binary alternatives.
Used in:API DesignAPI Design Best Practices
Retry-After
An HTTP header on 429 / 503 responses that tells clients exactly when to come back. Servers use it to actively shape incoming load; well-behaved clients stop retrying until the server invites them back.
Used in:Resilience Patterns
retry budget
A fleet-wide cap on retry traffic, typically expressed as a percentage of base requests. Past the budget, retries are dropped immediately even if individual policies would allow them. The lever that actually stops storms.
Used in:Resilience Patterns
retry storm
When clients retry failed requests aggressively against a downstream that's already overloaded, inflating offered load 2–3× exactly when capacity is shrinking. A self-inflicted denial-of-service.
Used in:Resilience Patterns
reverse proxy
A box that sits in front of one or more backend servers and pretends to be the server to the client. Three jobs always show up here: TLS termination (decrypt at the edge), compression (gzip/Brotli), and caching (serve repeat responses without touching the backend). Often the same physical process as the load balancer (NGINX, Envoy, HAProxy), but the role is distinct: balancing spreads load, proxying wraps it.
Used in:The API Gateway Pipeline
RPC remote procedure call
A function-oriented API style: the client calls a method on a remote service as if it were local, with the framework hiding the network behind a generated stub. Modern incarnations include gRPC, Thrift, and Cap'n Proto. Faster on the wire than REST and strongly typed; less universal (no browser-native support) and more coupled.
Used in:API Design
RPS · QPS · TPS
Requests / queries / transactions per second — the standard throughput units. Pick the one that matches what the system produces; a peak number is what you size capacity against.
Used in:Throughput
S
safe method safe HTTP method
An HTTP method that doesn't change server state. GET, HEAD, and OPTIONS are safe; the others are not. Safe methods can be cached aggressively, prefetched by the browser, and hit speculatively without worrying about side effects.
Used in:API Design
sched_setaffinity
The Linux syscall that pins a thread to a subset of cores. Wrapped by pthread_setaffinity_np in C, os.sched_setaffinity in Python, etc. Per-thread; finer-grained than taskset.
Used in:Core Pinning
single-flight request coalescing
When many concurrent callers want the same missing thing, only one of them does the work; the others block until its answer is ready, then read it. Go's singleflight package is the canonical implementation.
Used in:Resilience Patterns
SMT simultaneous multi-threading · Hyper-Threading · HT
A second register file and program counter on a physical core, sharing all execution units, caches, and predictor. Two contexts in flight; ~1.2–1.4× throughput per pair on CPU-bound work, not 2×. SMT is the hardware shortcut around an OS context switch.

SoA struct of arrays
Column-major memory layout: x[..], y[..], z[..], name[..]. Best when a loop touches one field across many records (the typical pattern in columnar databases and game engines).

spatial locality
"If you touched address X, you'll touch X+1, X+2 soon." The property real programs have when they iterate arrays linearly. What makes prefetching pay off.
Used in:Cache Locality
speculative execution
Running instructions on the predicted side of a branch before the branch is resolved. If the prediction is right, the work counts; if wrong, the pipeline flushes and the speculative instructions are discarded.
Used in:Cache Locality & Branch Prediction
SRAM static RAM
Six transistors per bit, no refresh needed. Fast (~1 ns) and used for CPU caches; expensive enough that you only get megabytes of it on a chip, not gigabytes.

SSL Secure Sockets Layer
The deprecated predecessor to TLS, whose name still gets used colloquially ("SSL certificate" usually means "TLS certificate"). The last SSL version (3.0) was deprecated in 2015; modern HTTPS is TLS 1.2 or 1.3. Treat the name as historical and use TLS when precision matters.

stale-while-revalidate
A cache strategy that serves the stale value immediately while a single background task refreshes. No client ever waits for a cold rebuild; the cost of staleness is bounded to one refresh cycle.

T
tail at scale
Dean & Barroso's observation: in a fan-out system with N parallel calls, the page's latency is dominated by the worst tail of N. p99 of one call becomes p90 of a 10-way fan-out. Why microservices obsess over the tail.
Used in:Latency & Throughput
temporal locality
"If you touched address X, you'll touch X again soon." The property real programs have when they reuse the same lookup tables and hot variables. What makes caching pay off.
Used in:Cache Locality
throttle
A hard cap on incoming rate or concurrency, enforced at the gateway. Past the limit, requests are rejected with 429 Too Many Requests. Prevents overload by saying no at the door.
Used in:Resilience Patterns
TLS Transport Layer Security
The protocol that encrypts HTTP into HTTPS. Provides three guarantees: encryption (only endpoints can read), integrity (tampering is detected via MAC), and authentication (the server proves identity via a certificate). TLS 1.3 (2018) is the modern version — one round-trip handshake, mandatory forward secrecy, retired RSA key exchange.
Used in:HTTPS & The TLS Handshake
TLS termination
Decrypting incoming HTTPS at the reverse proxy and talking plain HTTP (or re-encrypted HTTP) to the backend. Centralises certificate management, removes TLS CPU from every backend, and makes content-aware routing possible — you can't inspect or route by URL while the bytes are still encrypted.
Used in:The API Gateway Pipeline
TLB translation lookaside buffer
The CPU's cache of virtual-to-physical address translations. Flushed on every cross-process context switch; misses cost ~10–100 ns each as the page-table walker re-resolves the mapping.
Used in:Context Switching
token bucket
A throttling algorithm: a bucket holds up to N tokens and refills at rate R per second. Each request consumes one token; an empty bucket means rejection. Allows bursts up to N, sustains R over time.
Used in:Resilience Patterns
transient failure
A failure that succeeds if you try again, with no human intervention — a dropped TCP packet, a brief partition, a process restarting. Distinct from permanent failures (malformed request, missing record) which won't get better.
Used in:Resilience Patterns
TTL time to live
A per-record cache lease, in seconds. After expiry the resolver must re-fetch from upstream; until expiry the cached answer is what gets served — even if the authoritative answer has already changed. The whole story of DNS propagation delay is "TTL hasn't expired yet."
Used in:DNS ResolutionDNS TTL & Cutovers
tunable consistency
A database feature where the consistency level is set per-query rather than per-cluster. Cassandra's ONE / QUORUM / ALL are the canonical example.
Used in:CAP Theorem
U
under-fetching
An API returning less than the client needs in one response, forcing it to make additional round-trips to gather everything. Common in REST for graph-shaped reads ("user → orders → items" is N+2 calls). Network round-trip latency dominates the user-visible time. GraphQL solves it by collapsing related fetches into one shaped response.
Used in:API Design
V
vector clock
A per-node counter array used to track causal ordering of writes across a distributed system. AP systems use vector clocks (or variants) to detect and resolve conflicts when partitions heal.
Used in:CAP Theorem
vnode virtual node
A position marker on a hash ring. Each physical server plants 150–256 vnodes scattered around the ring; the data still lives on one disk per server, but the small slices average out to balanced load and graceful failover. Not the same as a ZooKeeper znode.
Used in:Consistent Hashing
volatility line
The boundary in the memory hierarchy between memory that survives a power cut (SSD, HDD, tape) and memory that doesn't (registers, cache, DRAM). The most consequential place to ask "what does this system actually have if power dies right now?"
Used in:Memory Hierarchy
W
WAL write-ahead log
A durable log written before a change is applied to its data structures. Every commit appends to the WAL and fsyncs it, then the change is replayed onto the main store; the design lets a crashed database recover by re-reading the log.
Used in:Hardware Pressure
Web Worker · worker_threads
A separate OS thread with its own JavaScript context and event loop, communicating with the main thread by message-passing (no shared state). True parallelism for browser/Node.js JS — used for genuinely heavy CPU work that would otherwise block the loop.
Used in:The Event Loop
working set
The bytes a workload actually touches in a window of time — the part of memory that has to fit in the fast tier for the program to run at speed. If it spills out of L2, every iteration eats DRAM latency; if out of DRAM, the program crashes or starts swapping.
Used in:Hardware PressureMemory Hierarchy
Z
znode
An entry in ZooKeeper's filesystem-shaped tree. Two flags change behaviour: ephemeral (auto-deleted when the client disconnects) and sequential (appended with a globally-unique monotonic suffix). The chained-watch leader-election recipe uses both. Not the same as a vnode.
