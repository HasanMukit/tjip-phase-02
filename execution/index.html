Theme 02 — Execution model

Many things at once.

The OS abstractions over hardware cycles. Processes are memory boundaries; threads are execution cursors; the event loop is concurrency without parallelism. Each abstraction has a cost — this theme is where it shows up.

Class 01

Process vs Thread

A process is a memory boundary; a thread is an execution cursor. The thread's "private kit" lives in registers while running and in the PCB while paused.
